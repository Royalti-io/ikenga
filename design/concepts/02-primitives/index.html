One workhorse, six variants, three sizes.

Primary = the right hand — at most one per visible region. Secondary fills role-balance. Outline / ghost for tertiary. Destructive uses --danger. Note Theme C: primary is verdigris (identity) so destructive (oxblood) becomes the action color.

08 · Accordion
In Theme C (Bronze Shrine), --primary is verdigris — the patina that signals "lived-in identity." That means the decisive-action role moves to oxblood (--danger). Reply, Send, and Confirm therefore render in red. This is intentional.
No. Density is a global preference — applied to <html> via data-density. Mixing per-screen would feel arbitrary to the user.
Workspace tints are constant hues (Mail = amber, Outbox = ember, Studio = oxblood, Agents = verdigris) and adapt only to mode lightness. They never shift when you change theme — they're the user's muscle-memory for "where am I in the app."
