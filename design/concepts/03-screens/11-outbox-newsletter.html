The newsletter, with its own clock.

Newsletters share the email_drafts table but earn their own page for two reasons: a cooling period (the pipeline drops a draft, and Approve is locked for two hours so you actually read it) and a quality scorecard the pipeline writes into metadata — word count, claims verified, anti-patterns, freshness, CTA count. Both surface here. Below: approval queue, schedule view, sent + engagement, the source pipeline, and the cross-channel hand-offs.

UX behaviour · badge · landing · the outer/inner Sent split
three rules that govern how this screen is reached
Badge logic
Outbox's Newsletter sub-route badge counts ready-to-approve only — drafts where reviewable_after <= now() AND status ∈ (draft, pending_review). Cooling drafts are not actionable yet, so they don't inflate the badge. Hover tooltip carries the full breakdown:
"2 ready · 1 cooling · ready 10:24 · 4 scheduled · 1 alert"
A small red dot appears on the badge if any send in the last 30d breached bounce > 2% or complaint > 0.1%.
Smart landing
/outbox/newsletter doesn't sticky on the last visited tab. It chooses based on state:
  • ≥ 1 ready-to-approve → Approval queue
  • only cooling drafts → Approval queue, scrolled to cooling section + banner "next ready in <mm>"
  • empty queue → Schedule (avoid the sad empty state — show what's queued instead)
  • 1st or last business day of month → Sent · Charts (review time bias)
Sent lives here
There is no outer /outbox/sent tab — sending happens by channel, so reading sent does too. Each channel owns its own Sent view:
  • /outbox/newsletter/sent · table by default; ?view=charts for engagement / deliverability / drilldown.
  • /outbox/email/sent, /outbox/social/sent, /outbox/sequences/sent · channel-specific tables; charts where the data supports it.
Cross-channel "what shipped today across all surfaces" lives outside Outbox — likely a digest in PA's morning briefing or a future /reports/outbox. Outbox itself stays focused on the work-to-do, by channel.

DReject · inline panel with canned reasons

Parked · what these charts can't show yet (and why)
Time-of-day open heatmap. Listmonk's campaign endpoint returns totals only — not per-hour event buckets. Would need separate instrumentation against Listmonk's view-tracking endpoint and a new newsletter_events table keyed by send + hour. Not blocking; defer until we have a deliverability question that actually needs it.
A/B subject win-rate. Schema has subject_alt on newsletter_sends but no per-variant engagement columns (opens_count_a, opens_count_b, etc.). Listmonk supports A/B campaigns; we'd need to teach the poller to split raw_stats per arm, then add the columns. Park until the poller is touched.
Per-segment performance. Segment lives on email_drafts.metadata not on newsletter_sends. Doable today via join, but inconsistent — older drafts didn't always set it. Add a "segment health" tab once metadata.segment is populated for ≥80% of sends, then we can show L5 Operators vs L5 Winback vs Investor side-by-side.
Single-edition open curve over first 72h. Same blocker as the heatmap — need per-event timestamps. Listmonk's webhook has them; we'd need to subscribe to the click + view webhooks (not currently wired). Worth doing if/when bounce-shaped issues recur.

GHow a draft gets here · the newsletter pipeline
Every draft you see in the queue traces back to .company/content/newsletters/<date>-<slug>/ on disk and the queue script that inserts the row. Knowing the source is useful: when an anti-pattern fires, you know which markdown file fed it; when you reject, you know which next pipeline run will rebuild.

Investor update · differences from newsletter
same UI · different defaults
Recipient list Twenty CRM segment instead of Listmonk segment. List is small (~12 leads), so per-recipient personalisation is heavier — first name, last meeting date, last raised stage.
Channel Always Resend. Listmonk's bulk pattern is wasted on 12 sends and the per-message reputation matters more than the bulk-list model.
Subject No A/B alt — investor list is too small for the variance to matter. Single subject, edited tightly.
Scorecard Adds metric clarity (every claim has a number + source). Drops previously featured (every update is meant to be cumulative).
Cooling Same 2h. The investor update will be read by the most attentive audience you have — a typo here is more expensive than one in the newsletter.
Schedule Default Thu 14:00 Lagos (mid-week, mid-afternoon — best response window for high-trust audience). Tuesday slot reserved for the broad newsletter.
IEmpty + skipped-month states
Two states that look similar but mean different things. Empty = no draft this month yet (pipeline hasn't run). Skipped = you (or the pipeline) decided this month doesn't have enough content to ship — visible in /outbox/newsletter for the rest of the month so it doesn't surprise anyone, with a one-click "force a draft now" escape.
