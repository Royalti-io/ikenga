Install · package manager

A Tauri 2 app where adding capability isn't a one-click file copy — it's a conversation with Claude. Pick a source, scan the manifest the kernel will register, adjust your settings, then hand the install off. The kernel writes the deterministic record; Claude does the personalised acquisition. Workspace tint = settings (cool clay).

A · Package manager · Install + Installed views

Two views, anchored in one frame so you can scroll the whole package-manager surface in a single pass. The Install view is the journey — discover · inspect · act, three rows always visible (no wizard hiding context). The Installed view is the registry — what's currently active, plus uninstall. The same com.ikenga.iyke meta-package shows in both: in Installed as the top row (auto-installed), and as the iyke endpoint badge on every handoff (it's how Claude drives the running app to finish the install).

C · Annotations
⓵ THREE ROWS · ALWAYS VISIBLE
No wizard hiding context
Discover · Inspect · Act all stack on the same page. The user can re-pick a source at any time without losing the manifest read or the settings they've already filled. Ikenga sits sovereign over its domain — the whole surface stays put.
⓶ SELECTED CARD · CHIP-CARVE
Ember accent + diamond corner
A selected catalog card lights up with the --achievement ember border, a kola-amber wash, and a chip-carve diamond in the top-right corner — the same diamond appears in the prompt-box corners and the row-rule dividers. One geometric motif, used sparingly across the page.
⓷ PERMISSIONS · OBSERVATIONS
Not a checkbox list
The permissions table shows what the package declared — not a row-by-row consent flow. Trust is negotiated when the user opens the install in chat. Risk dots use the kola-amber → oxblood ramp; shell.execute is always high.
⓸ HANDOFF · "OPEN IN CHAT" DOMINATES
Ember on top · oxblood below
The right-hand action — Claude in-app — is the kola-amber primary. Copy Prompt is the oxblood secondary, for terminal users. Both emit identical prompts (buildInstallPrompt in lib/install/prompt.ts). Below them: a tiny iyke endpoint badge so the user can see exactly what Claude will drive.
⓹ STATES · NEVER COLLAPSE THE CHROME
Rows stay · body swaps
Empty / error / skill-only states never remove the row scaffolding. The frame keeps its three rows; only the body of the affected row swaps. Predictable layout = stable mental model.
⓺ SECRETS · MASKED + VAULT-SCOPED
Never logged · 24 h grace
secret-typed settings render as masked inputs and store under vault.keys namespaced to the package id. On uninstall, the kernel keeps vault entries for 24 h to make reinstall friction-free.
⓻ INSTALLED · IYKE FIRST
Built-in surfaced, not hidden
The com.ikenga.iyke meta-package always appears as the first row in the Installed view, marked built-in and locked from uninstall. Surfacing it makes the system legible — every other package's handoff goes through iyke.
⓼ UNINSTALL · ATOMIC + REVERSIBLE
Confirm bubble · not a modal
Uninstall surfaces a row-local confirm bubble (no modal). It tells the user exactly what gets unregistered (registries + counts), what gets dropped (namespaced sqlite), and what gets kept (vault keys, for 24 h). The kernel rolls back atomically if any registry errors during the unregister sweep.
