The pane is the unit of work.

Every workspace lives inside a pane. Every pane has the same chrome — tab strip, body, drop zones — and the same set of states: focused, idle, loading, empty, errored, working. This page locks those states before we paint a single full route.

Order rulesleft → right
1. Pinned tabs cluster left, drag-locked between each other.
2. Open tabs follow, draggable between themselves.
3. Drag a normal tab onto a pinned tab → no swap, snaps back.
4. Drop on "+" zone = open new tab to the right.
3c · Pane action cluster · refresh + split + close

Sitting flush against the right edge of every tab strip, after the "+" New-tab affordance, is a cluster of pane-level actions: refresh, split right, split down, and close. A 1px hairline divider separates the additive "+" (creates a new tab) from these structural/destructive actions (operate on the whole pane). One-to-one with src/shell/panes/pane-toolbar.tsx — the design and the implementation share the same affordances, icons, and shortcuts.

3b · Workspace tint · per-tab classification rule

Each tab declares its workspace via data-ws. In workspace-mixed strips, inactive tabs render a 1px hairline at ~22% workspace-tint alpha; the active tab's bottom-border picks up its own workspace colour rather than the pane's. Single-workspace strips suppress the hairline — the pane focus accent already announces the workspace, so identical hairlines would just be noise. Pinned icon-only tabs skip the hairline (already self-identifying). Caller opts in by adding .tabstrip-ws-mixed to .content-tabs.
