You are the Chief Financial Officer for Royalti.io and Dixtrit.media. Your primary responsibilities are multi-entity financial tracking, reconciliation, and reporting.

Core Responsibilities

  • Bank statement processing — parse Kuda (NGN), Mercury (USD), GTB (NGN), Verto (multi-currency) into the standard transaction schema

  • Transaction classification — apply business rules + learned Tyms-accountant patterns
  • Inter-company reconciliation — match transfers between Royalti.io ↔ Dixtrit.media ↔ Personal
  • Runway + burn analytics — monthly runway recalc, scenario modeling

.claude/hooks/memory-coldstart.sh

memory-coldstart.sh SessionStart

Loads MEMORY.md + recent agent diary entries into the new session's initial context. Sync execution — blocks session start until done (typically <200ms). Configured in .claude/settings.local.json under SessionStart.hooks[2].
JSON config · settings.local.json
{ "SessionStart": [ { "hooks": [ { "type": "command", "command": ".claude/hooks/env-setup.sh" }, { "type": "command", "command": ".claude/hooks/mattermost-notify.sh", "async": true }, { "type": "command", "command": ".claude/hooks/memory-coldstart.sh" } ] } ] }
Script body .sh · 42 lines · 1.8 KB
#!/usr/bin/env bash # memory-coldstart.sh — load MEMORY.md + recent diary entries # Hooked to SessionStart · sync · target <200ms. set -euo pipefail PROJECT_DIR="${CLAUDE_PROJECT_DIR:-$(pwd)}" MEMORY_FILE="${PROJECT_DIR}/MEMORY.md" DIARY_DIR="${HOME}/.claude/projects/-home-nedjamez-royalti-co/memory" if [[ -f "$MEMORY_FILE" ]]; then echo "<memory-coldstart>" cat "$MEMORY_FILE" echo "</memory-coldstart>" fi # Recent diary entries (last 7 days) find "$DIARY_DIR" -name "diary_*.md" -mtime -7 | head -10 | while read f; do echo "<diary file=\"$f\">" cat "$f" echo "</diary>" done
Implementation note: per the Claude Code docs, there is no claude CLI subcommand for listing or inspecting agents/skills/commands. The desktop app reads the file system directly:
  • .claude/agents/*.md + ~/.claude/agents/*.md — parse YAML frontmatter from each file
  • .claude/skills/*/SKILL.md + ~/.claude/skills/*/SKILL.md — same, plus enumerate sibling files for the supporting-files tree
  • .claude/commands/*.md + ~/.claude/commands/*.md — same parser as agents
  • .claude/settings.local.json + ~/.claude/settings.json — read JSON, group by event key, resolve command paths to script bodies

A Tauri command (claude_config_load) would scan these paths once on screen-mount and watch with fs_watch for live updates — same pattern as the existing claude-sessions integration. Project files override personal files of the same name; the design surfaces this via the scope chip (PROJ / PERS) and an "Override" badge when both exist.
